VALIDAR MODELOS DE ML

VALIDAR MODELOS DE ML

En este blog, te presentaré un resumen del artículo "How to Correctly Validate Machine Learning Models" de RapidMiner. Aquí verás la importancia de una validación precisa de modelos de machine learning, para garantizar resultados confiables.

Feature Selection

La validación de modelos en machine learning es crucial para determinar la precisión de un modelo antes de su implementación.

Conceptos básicos de modelos predictivos y validación

Un modelo predictivo es una función que asigna un conjunto dado de valores para las columnas x a los valores correspondientes correctos para la columna y. Encontrar tal función para un conjunto de datos dado se llama entrenar el modelo. Esta capacidad de generalizar desde situaciones conocidas a situaciones futuras desconocidas es la razón por la que llamamos a este tipo particular de modelo "predictivo".

Error de entrenamiento vs. Error de prueba

La precisión de un modelo se evalúa midiendo su error de clasificación. Esto se logra comparando las predicciones del modelo con los valores reales. Hay dos tipos de errores principales:

  • Error de entrenamiento: Se calcula en los mismos datos utilizados para entrenar el modelo.
  • Error de prueba: Se calcula en conjuntos de datos diferentes para el entrenamiento y la evaluación, evitando la superposición entre datos de entrenamiento y prueba.
Proceso de cálculo de errores

En RapidMiner, un proceso típico para calcular el error de entrenamiento incluye cargar un conjunto de datos, alimentarlo a un modelo y evaluar su precisión usando el operador "Performance". En el caso del error de prueba, dos conjuntos de datos disjuntos se utilizan para el entrenamiento y la evaluación del modelo. Este proceso permite obtener una estimación más precisa de cómo el modelo se desempeñará en situaciones futuras desconocidas.

Por qué deberías ignorar el error de entrenamiento

Usar el error de entrenamiento te proporciona una validación del modelo inexacta. En lugar de proporcionar una evaluación precisa de la capacidad de un modelo para generalizar, el error de entrenamiento puede ser engañosamente optimista.

  • Validación utilizando conjuntos de datos de retención (Hold-out): Es una buena práctica utilizar una parte de los datos disponibles para el entrenamiento y otra parte para probar el modelo. Esta parte de los datos utilizada para las pruebas también se llama conjunto de datos de retención. Una buena práctica es utilizar un 70% de tus datos para el entrenamiento y el 30% restante para las pruebas.
  • Cross-Validation: Con la validación cruzada k-fold, no solo creas múltiples muestras de prueba repetidamente, sino que divides el conjunto de datos completo que tienes en k partes disjuntas del mismo tamaño. Luego, entrenas k modelos diferentes en k-1 partes cada uno, mientras pruebas esos modelos siempre en la parte restante de los datos. Es un método sencillo que te garantiza que no haya superposición entre los conjuntos de entrenamiento y prueba. También garantiza que no haya superposición entre los k conjuntos de prueba, lo cual es bueno ya que no introduce ninguna forma de sesgo de selección negativo.

Contaminación accidental

La contaminación accidental se refiere a la introducción de información de datos de prueba en los datos de entrenamiento durante la validación cruzada, lo que puede llevar a una estimación optimista y errónea del rendimiento del modelo en situaciones reales.

Ejemplo 1: Contaminación a través de la Normalización

Normalizar datos antes de la validación cruzada puede filtrar información de los datos de prueba en los de entrenamiento. El enfoque correcto es normalizar dentro de cada iteración de la validación cruzada, evitando la contaminación.

Ejemplo 2: Contaminación a través de la Optimización de Parámetros

Optimizar los parámetros del modelo es similar a otro proceso de aprendizaje automático y debe validarse correctamente. Validar solo el resultado de la optimización de parámetros puede llevar a estimaciones excesivamente optimistas. El enfoque correcto implica una validación cruzada anidada, que considera el impacto de la optimización en los datos de prueba.

Ejemplo 3: Contaminación a través de la Selección de Características

La selección de características busca elegir las columnas más relevantes para el modelo, pero también debe validarse correctamente. Elegir características antes de la validación cruzada puede introducir filtración de información. La validación adecuada requiere una validación cruzada anidada, considerando el efecto de la selección de características en el rendimiento.

Consecuencias de la contaminación ambiental

Es fundamental comprender las graves consecuencias que la contaminación accidental puede tener en la validación de modelos de aprendizaje automático. Aunque es posible construir modelos aparentemente buenos sin una validación adecuada, estos modelos podrían resultar mediocres o incluso peligrosos en situaciones reales de producción.

  • Ejemplo Médico: Imagina un modelo que utiliza resonancias magnéticas para detectar células tumorales malignas. Con una validación contaminada, el modelo parece excepcionalmente preciso, pero en la producción real, se comete un error del 10%, lo que podría llevar a tratamientos incorrectos o no identificados.
  • Ejemplo Empresarial: Una empresa invierte $20 millones para implementar medidas basadas en un modelo de aprendizaje automático que prometía reducir la pérdida de clientes en $40 millones. Sin embargo, debido a la validación incorrecta, solo logra un ahorro de $10 millones, lo que resulta en una pérdida de $10 millones en lugar de una ganancia esperada de $20 millones.
Cómo evitar la contaminación accidental
  • Esquemas de validación adecuados como la validación cruzada: realizar una validación cruzada adecuada, considerando todas las transformaciones y optimizaciones alrededor de la selección del modelo.
  • Esquemas de validación modulares: utilizar esquemas de validación modulares para medir el impacto de los cambios.
  • Anidación multinivel: para medir el efecto de acciones fuera de la selección del modelo, se requiere anidar múltiples validaciones cruzadas.
  • Modelos de preprocesamiento: las plataformas visuales deben admitir aplicar el mismo preprocesamiento a datos de prueba como se hizo en los de entrenamiento. Esto es especialmente problemático para transformaciones complejas. Muchos productos no admiten modelos de preprocesamiento, lo que limita su utilidad en escenarios prácticos y no científicos.
