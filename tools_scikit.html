SCIKIT - LEARN

En esta entrada, profundizaré en una de las librerías esenciales en machine learning: scikit-learn. Podrás ver algunas de sus numerosas funciones y los parámetros que cada una ofrece.

train_test_split

La función se utiliza para dividir un conjunto de datos en conjuntos de entrenamiento y prueba de manera aleatoria. Parámetros para esta función:

  • arrays (obligatorio): arrays o matrices que se dividen en conjuntos de entrenamiento y prueba.
  • test_size (opcional): especifica la proporción del conjunto de datos que se asignará al conjunto de prueba. /li>
  • train_size (opcional): permite especificar la proporción del conjunto de datos que se asignará al conjunto de entrenamiento.
  • random_state (opcional): indica la semilla utilizada por el generador de números aleatorios para garantizar que la división de datos sea reproducible.
  • shuffle (opcional): indica si los datos deben mezclarse antes de realizar la división.

LinearDiscriminantAnalysis

La función implementa el algoritmo de Análisis Discriminante Lineal (LDA). Parámetros para esta función:

  • solver (opcional): especifica el algoritmo utilizado para resolver el problema de optimización. Puede ser “svd” (por defecto), “lsqr” o “eigen”.
  • shrikage (opcional): controla si se aplica la regularización de Ledoit-Wolf.
  • n_components (opcional): especifica el número de componentes (o discriminantes lineales) a extraer.

LogisticRegression

La función se utiliza para crear un modelo de regresión logística. Parámetros para esta función:

  • penalty (opcional): especifica la norma de penalización utilizada en la regresión logística. Los valores son: “l1”, “l2”, “elasticnet” o None (por defecto).
  • C (opcional): inverso de la fuerza de regularización. Valores bajos aumentan la regularización, valores altos disminuyen la regularización.
  • solver (opcional): algoritmo de optimización a utilizar. Puede ser “newton-cg”, “lbfgs”, “liblinear”, “sag”, o “saga”.
  • max_iter (opcional): número máximo de iteraciones para que el solucionador converja.
  • multi_class (opcional): controla cómo manejar problemas de clasificación multiclase.
  • random_state (opcional): semilla del generador de números aleatorios utilizado para la inicialización del modelo.
  • class_weight (opcional): permite asignar pesos personalizados para las clases.

fit

La función se utiliza para entrenar un modelo utilizando datos de entrenamiento. Dependiendo si se utiliza LDA o Regresión Logística, que parámetros adicionales se utiliza, hay algunos que tienen en común:

  • X (obligatorio): matriz de características de entrenamiento.
  • Y (obligatorio): vector de etiquetas de clase correspondiente a las muestras de entrenamiento.
  • solver (opcional): el algoritmo a utilizar para resolver el problema de optimización durante el ajuste.

predict

La función se utiliza para realizar predicciones utilizando un modelo entrenado. Esta función tiene un solo parámetro:

  • X (obligatorio): matriz de características de prueba para la cual se desean realizar las predicciones.

classification_report

La función se utiliza para generar un informe de clasificación que muestra varias métricas de evaluación del rendimiento de un modelo de clasificación. Parámetros para esta función:

  • y_true (obligatorio): array que contiene las etiquetas reales de las muestras en el conjunto de prueba.
  • y_pred (obligatorio): array que contiene las etiquetas predichas por el modelo para las mismas muestras anteriores.
  • labels (opcional): lista de etiquetas de clase usadas para construir el informe de clasificación.
  • target_names (opcional): lista de nombres de clases que se utilizarán para etiquetar las filas del informe de clasificación.
  • sample_weight (opcional): permite asignar un peso específico a las muestras en la evaluación de métricas.
  • digits (opcional): indica la cantidad de dígitos que se mostrarán después del punto decimal en el informe.

confusion_matrix

La función se utiliza para calcular y mostrar una matriz de confusión que resume el rendimiento de un modelo de clasificación en un conjunto de datos. Parámetros para esta función:

  • y_true (obligatorio): array que contiene las etiquetas reales de las muestras en el conjunto de prueba.
  • y_pred (obligatorio): array que contiene las etiquetas predichas por el modelo para las mismas muestras anteriores.
  • labels (opcional): lista de etiquetas de clase usadas para construir la matriz de confusión.
  • target_names (opcional): permite asignar un peso específico a las muestras al calcular la matriz de confusión.
